7) 部署到 GitHub Pages（分支发布）
7.1 Git 与 SSH 准备
git config --global user.name  "lizhien23"
git config --global user.email "你的GitHub邮箱或 noreply 邮箱"

# 已创建过 SSH Key 可跳过；否则：
ssh-keygen -t ed25519 -C "你的邮箱"

# 测试连接（若成功会提示 You've successfully authenticated）
ssh -T git@github.com

7.2 仓库 Pages 设置（分支发布）

仓库名：lizhien23.github.io（必须与用户名一致）
GitHub → 仓库 → Settings → Pages：

Source：Deploy from a branch

Branch：master

Folder：/(root)

7.3 一键部署脚本

在站点根目录创建 deploy.sh：

#!/bin/bash
echo "🚀 开始部署 Hexo 博客..."
npx hexo clean
npx hexo g
npx hexo d
echo "✅ 部署完成！访问：https://lizhien23.github.io"


赋予执行权限并部署：

chmod +x deploy.sh
./deploy.sh


成功后几分钟内即可在 https://lizhien23.github.io
 看到更新。若缓存未刷新，强制刷新或换浏览器测试。

8) 常见问题速查

EACCES: permission denied（npm 全局安装报错）
用 npx 调命令；或用 nvm 安装 Node 再 npm i -g hexo-cli。

Port 4000 has been used（端口占用）
lsof -i:4000 → kill -9 <PID>；或 npx hexo s -p 5000。

fatal: unable to access ... Recv failure: Connection reset by peer（https 不稳定）
将部署与 git clone 改为 SSH 地址：git@github.com:...。

Author identity unknown（Git 未配置用户信息）
git config --global user.name "xxx" + git config --global user.email "xxx"。

GitHub Actions 与分支部署冲突，日志 "The operation was canceled"
Pages → Build and deployment 选择 Deploy from a branch（master/root），仅用 hexo d。

404 页面
访问不存在路径或 about/index.html 未生成。确保 source/about/index.md 存在，并执行 clean → g → d 后再访问 /about/。

头像变灰
themes/cactus/_config.yml 中 logo.grayout: false。

改成白色背景
themes/cactus/_config.yml 中 colorscheme: white 或 light。

10) 目录结构速览

└─ deploy.sh                   # 一键部署脚本（可选）
