从配置文件开始：搭建我的 Hexo 学术主页

John Doe
2025-08-28
GitHub Pages, Hexo, Linux, 教程

说明：
本文把“从 0 到上线”的全部步骤整理成一份可直接复制的 Markdown 文档。
为了便于一次性复制，本文件的所有内容（包括命令和配置）都在同一个代码块中给出。
你只需按顺序执行命令、将配置粘贴到相应文件即可完成搭建。

──────────────────────────────────────────────────────────────────────────────

  1. 环境准备
    ──────────────────────────────────────────────────────────────────────────────

推荐用 nvm 安装 Node，避免 npm -g 权限问题

curl -fsSL https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

重新加载 shell（若你用 bash）

source ~/.nvm/nvm.sh

安装 LTS 版本 Node

nvm install –lts

验证

node -v
npm -v
git –version

提示：

  • 如果系统已安装 Node/npm，可跳过。
  • 为避免 “npm i -g” 的 EACCES 报错，优先使用 npx 调 Hexo 命令；或使用 nvm 安装的 Node 后再全局安装。

──────────────────────────────────────────────────────────────────────────────
2) 初始化 Hexo 站点
──────────────────────────────────────────────────────────────────────────────

mkdir my-site && cd my-site
npx hexo init
npm install

本地预览

npx hexo server

浏览器访问: http://localhost:4000

若端口被占用（Port 4000 has been used）：
lsof -i:4000 # 找到占用端口的进程 PID
kill -9 # 结束进程
npx hexo server -p 5000

──────────────────────────────────────────────────────────────────────────────
3) 站点配置（根目录 _config.yml）
──────────────────────────────────────────────────────────────────────────────

说明：
下面是一份可直接使用的根配置示例。请根据你的账号替换 repo/url 等字段。
将以下 YAML 完整覆盖保存到: my-site/_config.yml

———————— my-site/_config.yml 开始 ————————

基本信息

title: 木子知恩的气象窗
subtitle: 深度学习与气象应用

url: https://lizhien23.github.io

内容与资源

new_post_name: :title.md

分页

部署（使用 SSH 推送到 GitHub 仓库）

type: git
repo: git@github.com:lizhien23/lizhien23.github.io.git
branch: master

———————— my-site/_config.yml 结束 ————————

4) 安装并启用 cactus 主题（学术简洁风）
──────────────────────────────────────────────────────────────────────────────

cd themes

cd ..

确认根 _config.yml 中已有：

theme: cactus

──────────────────────────────────────────────────────────────────────────────
7) 部署到 GitHub Pages（分支发布：master/root）
──────────────────────────────────────────────────────────────────────────────

7.1 Git 与 SSH 准备
git config –global user.name “lizhien23”
git config –global user.email “你的 GitHub 邮箱或 noreply 邮箱”

若未创建过 SSH Key：

ssh-keygen -t ed25519 -C “你的邮箱”

测试连接（看到 You’ve successfully authenticated 即成功）

ssh -T git@github.com

7.2 仓库 Pages 设置
仓库名：lizhien23.github.io（必须与 GitHub 用户名一致）
GitHub → 仓库 → Settings → Pages：
- Source 选择：Deploy from a branch
- Branch 选择：master
- Folder 选择：/ (root)

7.3 一键部署脚本（推荐）
在 my-site 根目录创建 deploy.sh，内容如下：

#!/bin/bash
echo “🚀 开始部署 Hexo 博客…”
npx hexo clean
npx hexo g
npx hexo d
echo “✅ 部署完成！访问：https://lizhien23.github.io“

赋权并执行：
chmod +x deploy.sh
./deploy.sh

部署成功日志包含：
INFO Deploy done: git
稍等片刻访问：
https://lizhien23.github.io

──────────────────────────────────────────────────────────────────────────────
8) 常见问题速查
──────────────────────────────────────────────────────────────────────────────

• npm 全局安装报错：EACCES: permission denied
解决：优先用 npx；或使用 nvm 安装 Node 后再 npm i -g hexo-cli。

• 端口占用：Port 4000 has been used
解决：lsof -i:4000 → kill -9 ；或 npx hexo s -p 5000。

• 拉取/推送 https 失败：Recv failure: Connection reset by peer
解决：改用 SSH 地址（git@github.com:…）。

• Git “Author identity unknown”
解决：git config –global user.name / user.email。

• GitHub Actions 与分支发布冲突，日志显示 “The operation was canceled”
解决：仓库 Pages 改为 Deploy from a branch（master/root），只用 hexo d 部署。

• 打开页面 404
解决：路径不存在或 about/index.html 未生成；确保执行 clean → g → d 后访问 /about/。

• 头像变灰
解决：themes/cactus/_config.yml → logo.grayout: false。

• 改成白色背景
解决：themes/cactus/_config.yml → colorscheme: white（或 light）。

9) 日常工作流（建议固定成 3 步）
──────────────────────────────────────────────────────────────────────────────

1. 写文章 / 改配置

npx hexo new “文章标题”

2. 本地预览（可选）

npx hexo s

3. 一键上线

./deploy.sh

10) 目录结构速览（关键部分）
──────────────────────────────────────────────────────────────────────────────

my-site/
├─ _config.yml # 站点配置（标题/URL/主题/部署）
├─ themes/
│ └─ cactus/
│ └─ _config.yml # 主题配置（导航/样式/社交/Logo）
├─ source/
│ ├─ _posts/ # 文章（Markdown）
│ ├─ images/ # 站点级图片（头像、favicon 等）
│ └─ about/
│ └─ index.md # 关于我页面
├─ public/ # 生成的静态文件（自动生成）
└─ deploy.sh # 一键部署脚本（可选）
